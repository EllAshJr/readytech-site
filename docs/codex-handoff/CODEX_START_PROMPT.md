# Prompt To Start A New Codex Session

Read AGENTS.md first.

Then read docs/codex-handoff/README.md and follow its reading order.

You are helping continue the ReadyTech build.

Important context:

- ReadyTech is a business-uptime and connected-operations company.
- The site is a Node/Express/EJS app.
- The user develops in WSL Ubuntu.
- Do not overbuild.
- Preserve the Instant Estimate and Sales Coach systems.
- Use the tone from AGENTS.md.
- Give safe, copy/paste friendly patches.
- Always inspect before changing.
- Always tell the user how to test locally.
- Never commit .env, credentials, database URLs, tokens, backups, or node_modules.

Before making code changes, summarize:

1. What files you will inspect
2. What files you expect to change
3. How you will test
4. What command the user should run afterward
