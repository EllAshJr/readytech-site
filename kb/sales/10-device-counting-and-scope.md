# Device Counting & Scope Worksheet

**Audience:** Sales technicians
**Read time:** 6 minutes
**Use this when:** Turning discovery answers into a monitoring tier or project estimate.

> **Sales outcome:** Count consistently, document assumptions, and avoid underquoting.

## What counts as a monitored device

Count each separately unless engineering approves grouping:

| Device type | Examples |
|---|---|
| Internet edge | Router, firewall, cellular router |
| Network | Managed switch, Wi-Fi access point |
| Compute | Server, critical workstation, appliance |
| Operations | POS terminal, KDS, important printer |
| Site system | NVR/camera recorder, power controller, sensor gateway |

Do not count every personal phone or guest device. Count infrastructure and agreed business-critical endpoints.

## Restaurant count example

```text
1 firewall
2 switches
4 access points
3 POS terminals
2 kitchen printers
2 KDS screens
1 NVR
-----------------
15 monitored devices
```

This fits within a 20-device ReadyContinuity Pro starting scope, subject to supportability.

## VPN scope units

Record:

- Remote-user count
- Location/site count
- Tunnel count
- Firewall brands/models
- MFA requirement
- Monitoring requirement

## Wi-Fi scope units

Record:

- Square footage and floors
- Indoor/outdoor zones
- Existing AP count
- Expected peak concurrent devices
- Cabling availability
- Guest/staff segmentation

AP quantity remains an estimate until survey/design validation.

## Backup scope units

Record:

- Users
- Workstations
- Servers
- Current GB/TB
- Growth estimate
- Retention and frequency
- Migration amount

## Assumptions log

Every estimate should state unknowns, for example:

> “Estimate assumes four supported access points, usable cabling, one firewall, and business-hours access. Final scope follows onsite verification.”

## Quality check before quoting

- No duplicate device counts
- Locations listed separately
- Supported/unsupported status noted
- Unknown items called out
- Monitoring tier matches total
- Hardware and carrier costs separated
- Engineering escalation triggered when limits are exceeded
